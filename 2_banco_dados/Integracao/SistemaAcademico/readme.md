# Metodologia de Desenvolvimento

O desenvolvimento deste projeto seguirá o **modelo Cascata (Waterfall)**, dividido nas etapas de levantamento de requisitos, projeto, implementação, testes e manutenção.

Por se tratar de um projeto acadêmico desenvolvido por apenas uma pessoa, onde o mesmo indivíduo desempenha os papéis de cliente, analista, arquiteto, desenvolvedor e testador, optou-se pela utilização do modelo cascata. Nesse contexto, os requisitos são definidos antes do início da implementação, reduzindo a necessidade de mudanças durante o desenvolvimento e permitindo um planejamento mais estruturado.

Como ferramenta de apoio ao gerenciamento das atividades, será utilizado o **Kanban**, permitindo organizar as tarefas, acompanhar o progresso do projeto e controlar o desenvolvimento de forma visual.

---

# Etapas do Desenvolvimento

## 1. Levantamento de Requisitos

Nesta etapa será realizado o levantamento dos requisitos do sistema, definindo suas funcionalidades, regras de negócio, entidades e relacionamentos necessários para o desenvolvimento.

O objetivo é compreender completamente o problema antes do início da implementação.

---

## 2. Projeto

Nesta etapa será realizado o planejamento técnico do sistema.

Entre as atividades previstas estão:

- Planejamento do desenvolvimento;
- Organização das tarefas no Kanban;
- Modelagem das classes;
- Modelagem do banco de dados;
- Elaboração dos diagramas necessários;
- Definição da estrutura do projeto.

Ao final desta etapa, todo o desenvolvimento deverá estar planejado e documentado.

---

## 3. Implementação

Com o planejamento concluído, inicia-se o desenvolvimento do sistema.

Nesta etapa serão implementados:

- Banco de dados;
- Classes;
- Regras de negócio;
- Persistência de dados;
- Integração entre Python e SQLite;
- Demais funcionalidades previstas no projeto.

---

## 4. Testes

Após a implementação, serão realizados testes para verificar se o sistema atende aos requisitos definidos durante o levantamento.

Também serão identificados possíveis erros, inconsistências e comportamentos inesperados, realizando as correções necessárias.

---

## 5. Manutenção

Após a conclusão do desenvolvimento, o sistema poderá receber melhorias, otimizações, correções de bugs e refatorações sempre que necessário.

Esta etapa permite a evolução contínua do projeto conforme novos conhecimentos forem adquiridos ao longo do roadmap.

---

Embora atualmente existam metodologias mais flexíveis para o desenvolvimento de software, como métodos ágeis, o modelo cascata mostra-se adequado para este projeto, pois seus requisitos são definidos pelo próprio desenvolvedor e tendem a sofrer poucas alterações durante cada ciclo de desenvolvimento.

A utilização do Kanban complementa esse processo, proporcionando uma forma simples e organizada de acompanhar o progresso das atividades e facilitar o gerenciamento do projeto.
